If implementation requires information not explicitly contained within these specifications:

1. Use the most conservative interpretation.

# IMPLEMENTATION PRINCIPLES

1. Do not simplify features because they appear complex.
2. Do not replace specified behaviour with easier alternatives.
3. Do not remove features on mobile.
4. Preserve all functionality across desktop, tablet and mobile layouts.
5. Auto-Focus Squad System is a core defining feature and must be implemented exactly as specified.
6. Group Stage Scroll-Snap Navigation is a core defining feature and must be implemented exactly as specified.
7. Hero Player Navigation behaviour must be implemented exactly as specified.
8. Tournament Centre functionality must be implemented exactly as specified.
9. If data is unavailable, use graceful fallbacks rather than removing UI elements.
10. Maintain a premium sports-dashboard experience.
11. Prioritize usability over visual gimmicks.
12. Prioritize performance without reducing functionality.

---

# PROJECT OBJECTIVE

Build the definitive World Cup 2026 Squad Explorer and Tournament Companion application.

The application should combine:

* FIFA World Cup presentation
* Football Manager depth
* Transfermarkt-style exploration
* Modern sports dashboard usability

The application should become a comprehensive reference tool for:

* Players
* Teams
* Squads
* Groups
* Fixtures
* Standings
* Knockout rounds
* Clubs
* Leagues
* Statistics

The application should be useful both for:

* Casual fans
* Serious football followers

---

# PRIMARY GOALS

The application should optimize for:

1. Fast player discovery
2. Team exploration
3. Squad analysis
4. Tournament tracking
5. Club and league exploration
6. Statistical analysis
7. Mobile usability
8. Long-term maintainability
9. Visual clarity
10. Performance

---

# DESIGN PHILOSOPHY

The application should NOT feel like:

* A spreadsheet
* A news website
* A betting website
* A fantasy football application

* A premium football intelligence platform
* A football database
* A World Cup exploration tool
* A modern sports dashboard

# TEAM EXPLORER

The Team Explorer is the primary area of the application.

Every World Cup team must be accessible.

Each team has its own dedicated page.

---

# TEAM PAGE STRUCTURE

Every team page contains four tabs:

1. Overview
2. Squad
3. Fixtures
4. Statistics

All four tabs are mandatory.

---

# OVERVIEW TAB

Purpose:

Provide a high-level summary of the team.

Contains:

* Team Header
* Top 5 Hero Players
* Tournament Snapshot
* Team Strength Radar
* Recent Form
* Club Distribution
* League Distribution
* Squad Makeup

---

# TEAM HEADER

Must display:

* Country Flag
* Country Name
* FIFA Ranking
* Manager
* Average Age
* Squad Value
* World Cups Won

May additionally display:

* Clubs Represented
* Leagues Represented
* Youngest Player
* Oldest Player
* Most Valuable Player

---

# TOP 5 HERO PLAYERS

Purpose:

Highlight the strongest players within the squad.

Display:

* Large Photo
* Player Name
* Club
* Position
* Consensus Score

Exactly five hero cards must be shown.

---

# HERO PLAYER BEHAVIOUR

Clicking a Hero Card must:

1. Open Squad Tab
2. Locate Player
3. Scroll To Player
4. Focus Player
5. Open Active Profile Panel

The user must never manually search for the selected player after clicking.

---

# TOURNAMENT SNAPSHOT

Must display:

* Current Group
* Current Position
* Record
* Goal Difference
* Next Match
* Kickoff Time
* Broadcaster

Purpose:

Provide immediate tournament context.

---

# TEAM STRENGTH RADAR

Visual comparison chart.

Suggested dimensions:

* Attack
* Midfield
* Defence
* Goalkeeping
* Depth

Purpose:

Quick understanding of team strengths.

---

# RECENT FORM

Display:

Recent match outcomes.

Examples:

W W D W W

Purpose:

Quick team form reference.

---

# CLUB DISTRIBUTION

Number of players from each club.

Purpose:

Understand squad makeup.

---

# LEAGUE DISTRIBUTION

Display:

Number of players from each league.

Purpose:

Understand league representation.

---

# SQUAD MAKEUP

Display:

* Goalkeepers
* Defenders

* Forwards

* Average Age
* Average Height

Purpose:

Provide structural squad overview.

# SQUAD TAB

This is the most important section of the application.

It contains:

* Official Squad
* Auto-Focus Squad System
* Active Profile Panel
* Reserves
* Coaching Staff

---

# AUTO-FOCUS SQUAD SYSTEM

This is a core project-defining feature.

The system must:

* Display player cards in rows
* Detect active row
* Automatically open profile information
* Automatically close previous profile information

This feature must not be replaced with:

* Accordions
* Modals
* Static profiles
* Always-open profiles

---

* Photo
* Name
* Position
* Club Badge
* Club Name

Desktop:

4 cards per row

Mobile:

2 cards per row

---

# GROUP STAGE

Core feature.

Must support:

* Horizontal scrolling
* Drag scrolling
* Mouse-wheel scrolling
* Large arrow buttons
* Scroll snapping

The user may navigate using either arrows or scrolling.

Both methods are required.

---

# GROUP STAGE CARD CONTENT

Each group card contains:

1. Standings
2. Upcoming Fixtures
3. Qualification Status

In that order.

All information visible without leaving the card.

---

# SCROLL SNAP REQUIREMENT

Groups must snap cleanly.

Examples:

Group A
→ Group B
→ Group C

No partial resting positions allowed.

---

# KNOCKOUT STAGE

Desktop:

Full bracket visible.

Mobile:

Horizontal bracket navigation.

* Live
* Finished

states.

# SUCCESS CRITERIA

The application succeeds if users can:

1. Explore every World Cup squad.
2. Discover players quickly.
3. Understand tournament standings instantly.
4. Navigate groups and knockout rounds effortlessly.
5. Compare teams easily.
6. Explore clubs and leagues represented at the tournament.
7. Use the application comfortably on desktop and mobile.
8. Use the application as a long-term World Cup reference resource.
